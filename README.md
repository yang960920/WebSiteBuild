# 🚀 WebBuild.Pro 프리랜서 포트폴리오 웹사이트

이 프로젝트는 웹사이트 구축 외주를 담당하는 프리랜서를 위한 다중 페이지(Multi-page) 포트폴리오 웹사이트입니다. 방문자에게 신뢰를 주고 원활한 문의(Lead) 전환으로 이어지도록 설계되었습니다.

## 🛠 사용 기술 스택 (Tech Stack)

- **프레임워크:** [Next.js 14 (App Router)](https://nextjs.org/)
- **UI/스타일링:** [Tailwind CSS](https://tailwindcss.com/)
- **언어:** [TypeScript](https://www.typescriptlang.org/)
- **아이콘:** [Lucide React](https://lucide.dev/)
- **컴포넌트/디자인:** Mobile-First 기반 반응형 웹 디자인, Custom CSS Variables (다크모드 지원)

## 📁 주요 페이지 구성 및 라우팅

웹사이트의 목적에 맞게 세분화된 8개 이상의 다중 페이지로 구성되어 있습니다.

1. **홈 (`/`)** : 히어로 섹션, 핵심 포트폴리오, 서비스 요약, 작업 프로세스, FAQ 및 전환 유도.
2. **서비스 (`/services`)** : 스타터, 비즈니스, 프리미엄 등 패키지별 상세 스펙과 제공 범위 안내.
3. **포트폴리오 목록 (`/portfolio`)** : 그동안 작업한 결과물 리스트 (기획, 디자인, 개발 등 카테고리 필터링 고려).
4. **포트폴리오 상세 (`/portfolio/[slug]`)** : 문제 제기 → 해결 과정 → 비즈니스 성과 구조의 템플릿 페이지.
5. **진행 방식 (`/process`)** : 체계적인 작업 타임라인 및 고객 준비물, 수정 정책 안내 (분쟁 방지용).
6. **가격/견적 (`/pricing`)** : 기준 견적표 및 투명한 결제 방식, 추가 비용 발생 조건 안내.
7. **만든이 소개 (`/about`)** : 개발자/디자이너로서의 강점, 커뮤니케이션 원칙, 기술 스택 소개.
8. **문의접수 (`/contact`)** : 예산, 마감일, 필수 기능 등을 종합적으로 확인하는 견적 문의 폼.
9. **기타 정책 (`/privacy`, `/terms`)** : 홈페이지 하단부에 노출되는 개인정보처리방침 및 이용약관.

## ⚙️ 로컬에서 실행하는 방법

사용자의 환경에 **[Node.js](https://nodejs.org/) (버전 18.17 이상)** 가 설치되어 있어야 합니다.

1. 터미널(가급적 PowerShell 또는 Bash)을 열고 현재 프로젝트 폴더로 이동합니다.
2. 의존성 패키지 설치를 위해 다음 명령어 중 하나를 실행합니다:
   ```bash
   npm install
   # 또는
   yarn install
   ```
3. 다음 명령어로 로컬 개발 서버를 시작합니다:
   ```bash
   npm run dev
   # 또는
   yarn dev
   ```
4. 터미널에 나타나는 로컬 주소(보통 `http://localhost:3000`)를 브라우저에서 열어 웹사이트를 확인합니다.

## ✨ 데이터 변경 및 유지보수 안내

- **포트폴리오 내용 수정:** `src/data/portfolio.json` 파일의 데이터를 편집하면 `/portfolio` 및 상세 페이지에 자동 반영됩니다.
- **연락처/이메일 수정:** 컴포넌트 내부 및 푸터(`src/components/Footer.tsx`)에 하드코딩된 이메일과 전화번호를 실제 정보로 교체하세요.
- **이미지 변경:** `public/images/` 디렉토리에 사용할 이미지를 넣고, 각 컴포넌트의 `src` 속성을 변경하시면 됩니다. (현재는 기본 플레이스홀더를 사용하도록 처리되어 있습니다.)

Github 저장소와 연동하여 버튼 클릭 몇 번으로 즉시 전 세계에 배포됩니다.
